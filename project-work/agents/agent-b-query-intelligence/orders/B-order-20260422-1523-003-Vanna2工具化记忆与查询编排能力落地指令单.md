# B-order-20260422-1523-003-Vanna2工具化记忆与查询编排能力落地指令单

## 1. 指令主题

在已完成 `vanna==2.0.2` 兼容升级的基础上，把 Vanna 2 真正值得用的新能力落到查询智能层。

本轮不做全量重写，重点落：

1. tool-oriented query orchestration
2. query memory / example retrieval explainability
3. audit-ready query context

## 2. 为什么由你负责

这些能力都属于查询智能 owner 范围，不属于 runtime 和前端。

你已有上下文连续性：

1. `vanna_doris.py`
2. 自然语言查询主链路
3. history / example retrieval / trace
4. query intelligence contract tests

## 3. 本次要落实的“升级后新能力”

### A. Tool 化编排最小落地

利用 Vanna 2 的 tool/adapter 思路，把现有自然语言查询主链路收口成更清晰的内部编排，而不是只停在 legacy base import 兼容。

要求：

1. 内部查询主链路拆出明确阶段：
   - memory retrieval
   - ddl/doc retrieval
   - sql generation
   - validation / repair
2. 允许使用 `LegacyVannaAdapter` 思路或等价内部适配层
3. 对外 API 契约保持稳定

### B. Query memory 可解释化

当前 query history / example retrieval 已存在，但解释面不够稳定。

要求：

1. 在查询结果 trace 里明确返回：
   - 命中的 example 数
   - 命中的 ddl/doc 数
   - retrieval source labels
2. 对 memory hit / fallback path 给出稳定字段，不要只靠日志
3. 尽量复用现有 Doris 向量检索能力，不重建第二套 memory 底座

### C. Audit-ready query context

利用 Vanna 2 user-aware / audit 思路，给现有 query path 加最小可扩展的上下文壳。

本轮目标：

1. 给自然语言查询链路引入稳定的 request/session/context envelope
2. 不强行做全套 RBAC
3. 先把“可被审计、可扩展、可传递”的结构壳搭起来
