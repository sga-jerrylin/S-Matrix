# 路线图

## Phase 0：项目启动准备

目标：

- 明确项目定位、范围和一期目标
- 补齐项目基础文档
- 统一当前代码基线认知

产出：

- 项目概述
- 当前状态
- 路线图
- 任务清单
- 工作约定

## Phase 1：基础链路打通

目标：

- 能完成环境启动
- 能导入样例数据
- 能跑通基础查询

重点：

- Docker 启动流程稳定
- README 与真实启动方式一致
- 样例数据和最小使用路径可验证

## Phase 2：智能查询能力收敛

目标：

- 自然语言查询主链路稳定
- 关键提示词、元数据、检索逻辑可解释
- 典型查询场景能复现

重点：

- 文本转 SQL 效果验证
- 多表查询体验
- 错误修复与失败回退

## Phase 3：工作台体验完善

目标：

- 前端工作台可支撑日常操作
- 上传、配置、查询、分析链路清晰

重点：

- 页面结构整理
- API 调用一致性
- 错误提示与状态反馈

## Phase 4：协同洞察能力

目标：

- 数据深挖不再只依赖内部表数据
- 能引入外部证据和外部趋势作为解释辅助

重点：

- 接入 `SGA-Web` 作为外部信息获取层
- 接入 `SGA-EastFactory` 作为外部趋势/舆情预测协同层
- 建立内部指标异常 -> 外部证据补充 -> 结论生成的闭环

## Phase 5：预测能力落地

目标：

- 形成可用的业务预测主链路
- 支持时间序列预测、外生变量、回测和情景模拟

重点：

- 先做内部业务指标预测
- 让 `EastFactory` 输出可作为外生变量接入
- 让贝叶斯优化服务于模型和策略搜索，而不是直接充当产品主角

## Phase 6：CLI 与能力输出

目标：

- 形成稳定 CLI 能力面
- 可被 `SGA-Molt` skills 和 MCP 调用

重点：

- `query / insight / forecast / report / mcp serve`
- 统一 JSON 输出和错误码
- 固化本地与远程两类集成方式

## Phase 7：交付与稳定性

目标：

- 形成可演示、可交接、可继续开发的基线

重点：

- 文档收口
- 验证脚本收口
- 风险清单和后续规划收口

## 当前建议优先级

1. 先把 Phase 0 做扎实
2. 再验证 Phase 1 主链路
3. 尽快明确 Phase 4-6 的边界，不要让 `DorisClaw` 吸收其他项目的职责
