# 项目概述

## 项目名称

DorisClaw

## 项目简称

DC

## 名称说明

- `DorisClaw` 是当前确认的未来正式名称
- `S-Matrix` 视为仓库历史阶段名称
- 后续对外规划、CLI、文档和集成协议优先使用 `DorisClaw / DC`

## 项目定位

一个基于 Apache Doris 的数据决策能力层，重点解决“让用户对结构化业务数据完成查询、洞察和预测”的问题。

## 核心目标

- 降低结构化数据查询门槛
- 让非 SQL 用户也能完成数据检索与分析
- 把上传、建表、查询、分析、配置放到一个统一工作台中
- 为后续的协同洞察与业务预测打下统一底座
- 形成可被 `SGA-Molt` skills 调用的稳定 CLI 能力面

## 核心能力

- Excel 上传与数据导入
- 数据表管理与元数据分析
- 自然语言转 SQL
- 多表查询规划与结果协调
- SQL 失败后的自动修复
- 数据深挖与分析报告
- 业务预测与情景模拟
- LLM 配置管理
- 数据源同步
- CLI / MCP 能力输出

## 目标用户

- 业务分析人员
- 运营人员
- 需要快速看数但不擅长 SQL 的使用者
- 需要搭建内部智能数据查询入口的团队
- 需要把数据能力嵌入 agent / skills / workflow 的上游系统

## 当前项目范围

现阶段先聚焦以下范围：

- 本地或内网环境可部署
- 数据上传与查询主链路可跑通
- 前后端基础工作台可使用
- 自然语言查询链路可验证
- 数据深挖链路可结合外部上下文协同解释
- 业务预测链路可使用时间序列和外部信号
- 文档、配置、运行方式可交接

## 当前不优先的范围

- 大规模多租户 SaaS 化
- 复杂权限中心
- 商业化计费体系
- 完整企业级运维平台
- 过早的多团队协作机制
- 自建完整外部搜索与抓取底座
- 自建完整外部舆情预测平台
- 自建算法进化平台

## 现阶段成功标准

- 新人拿到仓库后能完成基础启动
- 能上传一份样例数据并完成查询
- 能对样例数据完成一次数据深挖
- 能对至少一个业务指标完成一次预测
- 能看懂项目结构、当前状态和下一步安排
- 后续开发不再从零整理上下文
